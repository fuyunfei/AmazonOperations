## 各层职责边界

| 层 | 职责 | 不做 |
|----|------|------|
| `app/api/` | 接收请求、调用 lib、返回 JSON/SSE | 不含业务逻辑 |
| `lib/parsers/` | XLSX buffer → 结构化 Row[] | 不写 DB，不知道 fileType 以外的上下文 |
| `lib/rules/` | Row[] → ActionItem[] / Alert[] | 不查 DB，不知道 HTTP 请求 |
| `lib/config.ts` | 读取三层参数配置 | 不修改配置 |
| `components/panels/` | 展示数据，调 API | 不含业务逻辑 |
| `store/` | 前端导航状态 | 不缓存业务数据（每次从 API 取最新） |

## 文件类型枚举

所有报表在系统内用以下 `fileType` 标识，贯穿 parser / DB / API / Chat 工具：

| fileType | 报表名 | 存储策略 | Parser 状态 |
|----------|--------|---------|------------|
| `product` | 产品报表-子ASIN视图 | `ProductMetricDay`（时序累积） | ✅ `parseAsinReport` |
| `search_terms` | 搜索词重构（US，30天） | `ContextFile`（快照覆盖） | ✅ `parseSearchTerm` |
| `campaign_3m` | 广告活动重构（ALL，3月） | `ContextFile`（快照覆盖） | ✅ `parseAdRestructure` |
| `us_campaign_30d` | US广告活动（活动视图，30天） | `ContextFile`（快照覆盖） | ✅ `parseAdCampaign` |
| `placement_us_30d` | 广告位报表（US，30天） | `ContextFile`（快照覆盖） | ✅ `parseAdPlacement` |
| `inventory` | 库存报表 | `ContextFile`（快照覆盖） | ✅ `parseInventory` |
| `keyword_monitor` | 关键词监控 | `ContextFile`（快照覆盖） | ✅ `parseKeywordMonitor` |
| `cost_mgmt` | 成本管理 | `ContextFile`（快照覆盖） | ✅ `parseCostMgmt` |
| `aba_search` | ABA搜索词对比 | `ContextFile`（快照覆盖，暂不使用） | ✅ `parseAbaSearch` |
