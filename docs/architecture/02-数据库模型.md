## 表一：ContextFile（快照类报表）

```prisma
model ContextFile {
  id           String   @id @default(cuid())
  fileType     String   @unique   // 见 fileType 枚举表
  fileName     String             // 原始文件名，如 "search_terms_30d.xlsx"
  uploadDate   DateTime @default(now())
  snapshotDate String             // 来自报表 D2 字段，如 "2026-04-08"
  parsedRows   Json               // Row[] 直接序列化为 JSON
}
```

**存储策略**：按 `fileType` upsert。每次上传同类型报表，整条记录被替换（保留最新版本）。

**包含的报表**：除产品报表以外的全部 8 种报表类型。

**`parsedRows` 字段结构**：各报表解析后字段名见 `功能数据流/04-广告优化行动清单.md` 中"处理的数据字段"一节。示例：

```json
// fileType = "search_terms" 时的 parsedRows 示例
[
  {
    "asin": "B0GD7BF2TZ",
    "search_term": "foam mattress thick",
    "match_type": "广泛",
    "impressions": 1240,
    "clicks": 22,
    "spend": 31.4,
    "ad_orders": 0,
    "ad_sales": 0,
    "ctr": 0.0177,
    "cpc": 1.43,
    "acos": null,
    "conversion_rate": 0,
    "campaign": "SP-BF2TZ-Broad-Sofa",
    "placement_type": "SP手动"
  }
]
```

## 表二：ProductMetricDay（产品报表时序）

```prisma
model ProductMetricDay {
  id      String @id @default(cuid())
  asin    String
  date    String   // "2026-04-08"（来自报表 D2 字段的结束日期）
  metrics Json     // 当日所有指标字段

  @@unique([asin, date])   // 同日期重复上传时 upsert，不重复堆积
}
```

**存储策略**：按 `(asin, date)` upsert。每日上传产品报表时，解析所有 ASIN 行，逐行 upsert 到此表。已有的日期数据会被同日期的新上传覆盖，不同日期数据永远保留。

**与 ContextFile 的区别**：

| | ContextFile | ProductMetricDay |
|--|-------------|-----------------|
| 上传新文件时 | 整条记录替换 | 按 (asin, date) 追加/更新 |
| 历史数据 | 不保留 | 永久保留 |
| 用途 | 最新快照供规则引擎使用 | 时序计算（环比、日均、趋势） |

**`metrics` 字段结构**：

```json
{
  "gmv": 218.5,
  "orders": 2,
  "units": 2,
  "returns": 0,
  "ad_spend": 45.2,
  "ad_sales": 109.0,
  "ad_orders": 1,
  "impressions": 3200,
  "clicks": 28,
  "sessions": 112,
  "ocr": 0.018
}
```

> **存储原则**：只存原始计数，不存比率字段（ACOS / CTR / CVR 等由查询时动态计算，保证全系统唯一计算口径）。  
> **ad_spend 符号**：Nordhive 报表中广告花费为负数（记账方向），parser 解析时取绝对值存入（`ad_spend = Math.abs(rawValue)`）。

**常用查询示例**：

```ts
// 近 7 天某 ASIN 的日均 GMV
const rows = await db.productMetricDay.findMany({
  where: { asin: "B0GD7BF2TZ", date: { gte: "2026-04-05" } },
  orderBy: { date: "asc" }
})
const avgGmv = rows.reduce((s, r) => s + (r.metrics as any).gmv, 0) / rows.length

// 环比：今日 vs 7 天前同日
const today    = await db.productMetricDay.findUnique({ where: { asin_date: { asin, date: "2026-04-09" } } })
const weekAgo  = await db.productMetricDay.findUnique({ where: { asin_date: { asin, date: "2026-04-02" } } })
const sessionsChange = ((today.metrics.sessions - weekAgo.metrics.sessions) / weekAgo.metrics.sessions)
```

## 表三：CategoryMap（品类-ASIN 映射）

```prisma
model CategoryMap {
  id          String @id @default(cuid())
  categoryKey String @unique   // "mattress" | "pump" | "scooter"
  displayName String           // "床垫" | "充气泵" | "电动滑板车"
  asins       Json             // string[]，如 ["B0GD7BF2TZ", "B0GD7K1TC9"]
}
```

**存储策略**：手动维护，通过 `prisma/seed.ts` 初始化。系统启动时不自动修改。

**Seed 数据**（`prisma/seed.ts`）：

```ts
await db.categoryMap.createMany({
  data: [
    { categoryKey: "mattress", displayName: "床垫",     asins: JSON.stringify(["B0GD7BF2TZ", "B0GD7K1TC9"]) },
    { categoryKey: "pump",     displayName: "充气泵",   asins: JSON.stringify(["B0FJKNRNZK"]) },
    { categoryKey: "scooter",  displayName: "电动滑板车", asins: JSON.stringify([]) },
  ],
  skipDuplicates: true,
})
```

**用途**：
- 品类视图页面确定展示范围
- 规则引擎过滤 `parsedRows` 只处理本品类 ASIN
- 跨 ASIN 词重叠检测（P2 内部竞争）

## 表四：AsinConfig（ASIN 级别配置）

```prisma
model AsinConfig {
  id          String   @id @default(cuid())
  asin        String   @unique
  categoryKey String              // 与 CategoryMap.categoryKey 对应
  stage       String              // "launch" | "growth" | "mature"
  displayName String?             // 可选展示名，如 "Queen 4\" 沙发床垫"
  updatedAt   DateTime @updatedAt
}
```

**用途**：存储每个 ASIN 的产品阶段，供告警规则（03）按阶段选取不同 ACoS 阈值。

**与 CategoryMap 的关系**：
- `CategoryMap.asins` 是快速 ASIN 列表（用于过滤查询）
- `AsinConfig` 是 ASIN 的详细配置（用于规则引擎取阶段参数）
- 新增 ASIN 时，两处都需更新

**Seed 数据**（`prisma/seed.ts`）：

```ts
await db.asinConfig.createMany({
  data: [
    { asin: "B0GD7BF2TZ", categoryKey: "mattress", stage: "growth", displayName: "Queen 4\" 沙发床垫" },
    { asin: "B0GD7K1TC9", categoryKey: "mattress", stage: "growth", displayName: "Full 4\" 沙发床垫"  },
    { asin: "B0FJKNRNZK", categoryKey: "pump",     stage: "launch", displayName: "150PSI 充气泵"      },
  ],
  skipDuplicates: true,
})
```

**阶段查询示例**：

```ts
// 规则引擎取某 ASIN 的告警阈值
const config    = await db.asinConfig.findUnique({ where: { asin } })
const acosRed   = getParam("acos_red", config.categoryKey, config.stage)
// → mattress + growth → category override → 0.60
```

## 表五：Alert（每日告警，ASIN × 指标粒度）

```prisma
model Alert {
  id           String   @id @default(cuid())
  asin         String
  categoryKey  String                    // "mattress" | "pump"
  metric       String                    // "acos_red" | "sessions_drop" | "inventory_low" | "ctr_low"
  level        String                    // "red" | "yellow"
  currentValue Float                     // 实际值，如 0.824（ACOS 82.4%）
  threshold    Float                     // 触发阈值，如 0.55
  stage        String                    // "launch" | "growth" | "mature"（确定适用哪套阈值）
  suggestion   String                    // 建议操作文本
  snapshotDate String                    // 产品报表日期，如 "2026-04-09"
  createdAt    DateTime @default(now())

  @@unique([asin, metric, snapshotDate])
  @@index([categoryKey, snapshotDate])
}
```

**存储策略**：每次上传产品报表后，按 `snapshotDate` 全量重写——先 `deleteMany({ where: { snapshotDate } })` 删除当日旧记录，再批量插入新结果。指标恢复正常后自然不再出现在列表中，无需"已处理"状态字段。

**写入时机**：POST /api/upload 末尾，仅当上传的 fileType 属于告警引擎依赖文件时触发（product / keyword_monitor / inventory / us_campaign_30d）。见 `项目结构/03-数据流.md`。

**Alert 表是必需的**：Chat 工具 `get_alerts()` 直接查询此表。

## 表六：Session（Chat 会话）

```prisma
model Session {
  id        String    @id @default(cuid())
  title     String    @default("新对话")   // 默认取首条用户消息前 30 字
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt
  messages  Message[]
}
```

**存储策略**：用户点击「新建对话」时创建。删除 Session 时级联删除所有关联消息。

## 表七：Message（Chat 消息）

```prisma
model Message {
  id         String   @id @default(cuid())
  sessionId  String
  session    Session  @relation(fields: [sessionId], references: [id], onDelete: Cascade)
  role       String   // "user" | "assistant"
  content    String   // 用户可见的文字内容
  toolCalls  Json?    // 本轮工具调用摘要（可选，用于 UI 展示）
  createdAt  DateTime @default(now())
  tokenCount Int?     // 本轮消耗 token（可选，用于用量统计）

  @@index([sessionId, createdAt])
}
```

**`toolCalls` 字段结构**（存摘要，不存完整返回 JSON）：

```json
[
  { "tool": "get_search_terms", "input": {"filter": "high_acos"}, "result": "返回 22 条零成交高点击词" },
  { "tool": "get_ad_campaigns", "input": {"filter": "high_acos"}, "result": "5 个活动，最高花费 $889" }
]
```

**加载策略**：`GET /api/sessions/:id` 取最近 40 条消息（= 最近 20 轮 user+assistant 对）。Session 历史过长时自动截断早期记录，防止 context 超限。

## 数据库表总览

| # | 表名 | 存储内容 | 写入策略 | MVP状态 |
|---|------|---------|---------|---------|
| 1 | `ContextFile` | 快照类报表解析结果 | upsert by fileType | ✅ 实现 |
| 2 | `ProductMetricDay` | 产品报表每日指标 | upsert by (asin, date) | ✅ 实现 |
| 3 | `CategoryMap` | 品类 → ASIN 列表 | seed 手动维护 | ✅ 实现 |
| 4 | `AsinConfig` | ASIN → 品类 + 阶段 | seed 手动维护 | ✅ 实现 |
| 5 | `Alert` | 每日告警快照 | delete+insert by snapshotDate | ✅ 实现 |
| 6 | `Session` | Chat 会话元数据 | create / update / delete | ✅ 实现 |
| 7 | `Message` | Chat 消息历史（含工具调用摘要） | create（每轮 done 后写入） | ✅ 实现 |
