## 概览

```
用户操作                 服务端                        数据库
─────────               ──────────────────────         ──────────────
上传 XLSX
  │
  ▼
POST /api/upload   →   identifyFileType(filename)
                       ├─ product      → parseProduct(buffer)       → upsert ProductMetricDay (asin × date)
                       └─ others       → contextParsers[type](buffer) → upsert ContextFile (by fileType)
                       │
                       保存原始文件到 context/ 目录
                       │
                       返回 { fileType, snapshotDate, rowCount }
  │
  ▼
Context 面板刷新
页面触发 features API

GET /api/features/*  →  读 CategoryMap（确定 ASIN 范围）
                        读 ContextFile（快照数据）
                        读 ProductMetricDay（时序数据）
                        JOIN / 过滤 / 聚合
                        lib/rules/*(input) → 结果数组
                        返回 JSON
  │
  ▼
Panel 渲染
```

## 一、文件上传流程

### 1.1 前端触发

```
用户在 Context 面板点击「+ 添加文件」
  → <input type="file" accept=".xlsx"> 弹出文件选择框
  → 选择文件后，读取 File 对象
  → 根据文件名推断 fileType（或展示下拉让用户确认）
  → POST /api/upload  multipart/form-data
      { file: File, fileType: string }
```

**fileType 推断规则**（`lib/parsers/identifier.ts`，顺序敏感，越具体越靠前）：

| 匹配条件（文件名 toLowerCase） | 推断 fileType |
|-------------------------------|--------------|
| 含 `产品报表` **且** 含 `asin` | `product` |
| 含 `广告活动` **且** 含 `重构` **且** 含 `all` | `campaign_3m` |
| 含 `搜索词重构` 或（含 `搜索词` 且含 `重构`） | `search_terms` |
| 含 `广告位` | `placement_us_30d` |
| 含 `广告活动` **且不含** `重构` | `us_campaign_30d` |
| 含 `库存报表` 或 含 `库存` | `inventory` |
| 含 `关键词监控` 或 含 `关键词` | `keyword_monitor` |
| 含 `成本管理` 或 含 `成本` | `cost_mgmt` |
| 含 `aba` | `aba_search` |
| 其他 | `unknown`（弹窗让用户手动确认） |

> **注意**：`campaign_3m` 必须同时包含"重构"和"ALL"，`us_campaign_30d` 则只需含"广告活动"且不含"重构"，两者匹配顺序不可颠倒。`product` 需含"ASIN"以排除 SKU 视图。

### 1.2 服务端处理（`/api/upload/route.ts`）

```ts
export async function POST(req: NextRequest) {
  const formData       = await req.formData()
  const file           = formData.get("file") as File
  const fileTypeOverride = formData.get("fileType") as string | null

  // 1. 推断 fileType（前端可传 fileType 覆盖自动推断结果）
  const fileType: FileType = fileTypeOverride ?? identifyFileType(file.name)
  if (fileType === "unknown")     return error("无法识别文件类型")
  if (!isParseableType(fileType)) return error("该类型暂无 parser")

  const buffer = Buffer.from(await file.arrayBuffer())

  // 2. 解析：product 和其他报表走不同路径
  if (fileType === "product") {
    const { rows, snapshotDate } = parseProduct(buffer)
    await Promise.all(
      rows.map(row =>
        db.productMetricDay.upsert({
          where:  { asin_date: { asin: row.asin, date: snapshotDate } },
          update: { metrics: JSON.stringify(row.metrics) },
          create: { asin: row.asin, date: snapshotDate, metrics: JSON.stringify(row.metrics) },
        })
      )
    )
  } else {
    // contextParsers[fileType](buffer, filename) → { rows, snapshotDate }
    // aba_search 的 snapshotDate 从文件名提取（无 Nordhive 日期行）
    const { rows, snapshotDate } = contextParsers[fileType]!(buffer, file.name)
    await db.contextFile.upsert({
      where:  { fileType },
      update: { fileName: file.name, snapshotDate, parsedRows: JSON.stringify(rows) },
      create: { fileType, fileName: file.name, snapshotDate, parsedRows: JSON.stringify(rows) },
    })
  }

  // 3. 保存原始文件到 context/ 目录
  const contextDir = path.join(process.cwd(), "context")
  await fs.mkdir(contextDir, { recursive: true })
  await fs.writeFile(path.join(contextDir, file.name), buffer)

  // 4. 触发告警引擎（product / keyword_monitor / inventory / us_campaign_30d 时才触发）
  await runAndPersistAlerts(fileType)    // → Alert 表

  return NextResponse.json({ fileType, snapshotDate, rowCount })
}
```

### 1.3 Parser 结构

**Nordhive 导出文件固定格式**（`lib/parsers/utils.ts` 的 `parseNordhiveSheet` 处理此结构）：

```
Row 1（idx 0）：筛选条件标签行（含"时间范围:"等标签）
Row 2（idx 1）：筛选条件值行（col[3] = 时间范围字符串，如 "2026-03-10 - 2026-04-09"）
Row 3（idx 2）：空行
Row 4（idx 3）：列名行（ASIN / 搜索词 / 匹配类型 / 曝光量/展示次数 ...）
Row 5（idx 4）：Total 汇总行（parser 跳过，不作为数据行）
Row 6+（idx 5+）：数据行
```

> **例外**：ABA 搜索词对比文件 Row 1 直接是列名，无前置元数据行，由 `parseFlatSheet` 处理。日期从文件名中提取（格式 `YYYYMMDD`）。

**公共工具函数**（`lib/parsers/utils.ts`）：

```ts
// 解析 Nordhive 格式 → { dateRange, headers, rows }
// dateRange 来自 raw[1][3]；列头取 raw[3]；数据从 raw[5] 开始（跳过 Total）
export function parseNordhiveSheet(ws: WorkSheet): { dateRange: string; headers: string[]; rows: RawRow[] }

// 从日期范围字符串推断 timeWindow
// "2026-03-10 - 2026-04-09" (31天) → "monthly"
// "2026-04-03 - 2026-04-09" (7天)  → "weekly"
export function inferTimeWindow(dateRange: string): { timeWindow: string; startDate: Date|null; endDate: Date|null }
```

**以搜索词重构为例**（`lib/parsers/parseSearchTerm.ts`）：

```ts
export function parseSearchTerm(buffer: Buffer): SearchTermRow[] {
  const wb = readWorkbook(buffer)
  const ws = wb.Sheets[wb.SheetNames[0]]
  const { dateRange, rows } = parseNordhiveSheet(ws)
  const { timeWindow } = inferTimeWindow(dateRange)

  return rows
    .filter((row) => row["搜索词"] != null && String(row["搜索词"]).trim() !== "")
    .map((row) => {
      const clicks  = toInt(row["点击量"])
      const impr    = toInt(row["曝光量/展示次数"])    // 注意列名含"/"
      const spend   = Math.abs(toNum(row["广告花费"]))  // Nordhive 花费为负数，取绝对值
      const sales   = toNum(row["广告销售额"])
      const orders  = toInt(row["广告订单量"])
      const acosRaw = toNum(row["ACoS"])

      return {
        asin:         String(row["ASIN"] ?? "").trim(),
        searchTerm:   String(row["搜索词"] ?? "").trim(),
        matchType:    String(row["匹配类型"] ?? "").trim(),
        bid:          toNum(row["竞价"]),
        campaignName: String(row["广告活动"] ?? "").trim(),
        adType:       String(row["投放类型"] ?? "").trim(),
        timeWindow,
        impressions: impr,
        clicks,
        spend,
        sales,
        orders,
        // 比率字段由 parser 计算（不依赖报表原始值），统一口径
        acos: acosRaw > 1 ? acosRaw : acosRaw * 100,  // 兼容小数和百分比
        roas: spend > 0 ? sales / spend : 0,
        ctr:  impr > 0 ? (clicks / impr) * 100 : 0,
        cpc:  clicks > 0 ? spend / clicks : 0,
        cvr:  clicks > 0 ? (orders / clicks) * 100 : 0,
      }
    })
}
```

**snapshotDate 提取**（`lib/parsers/index.ts`）：

```ts
// 从 D2 提取报表日期（独立于各 parser，避免重复逻辑）
function extractSnapshotDate(buffer: Buffer): string {
  const raw = sheetToRows(readWorkbook(buffer).Sheets[0])
  const dateRange = (raw[1]?.[3] as string) ?? ""
  const parts = dateRange.split(" - ")
  return (parts[1] ?? parts[0] ?? "").trim()  // 取结束日期
}

// contextParsers 统一包装：parser函数 + snapshotDate
export const contextParsers: Partial<Record<FileType, ContextParser>> = {
  search_terms: (buf) => ({ rows: parseSearchTerm(buf), snapshotDate: extractSnapshotDate(buf) }),
  campaign_3m:  (buf) => ({ rows: parseAdRestructure(buf), snapshotDate: extractSnapshotDate(buf) }),
  // ... 其他报表类似
  aba_search: (buf, filename = "") => {
    // ABA 无 Nordhive 日期行，从文件名 YYYYMMDD 提取
    const m = filename.match(/(\d{4})(\d{2})(\d{2})/)
    return { rows: parseAbaSearch(buf, filename), snapshotDate: m ? `${m[1]}-${m[2]}-${m[3]}` : today() }
  },
}
```

## 三、规则引擎结构

```ts
// lib/rules/sop/index.ts

interface SopInput {
  searchTerms:          SearchTermRow[]
  campaignWithBudget:   CampaignRow[]   // campaign_3m JOIN us_campaign_30d
  placement:            PlacementRow[]
  categoryAsins:        string[]
  config:               typeof getParam
}

export function runSopRules(input: SopInput): ActionItem[] {
  return [
    ...applyP0Rules(input),
    ...applyP1Rules(input),
    ...applyP2Rules(input),
    // ...applyP3Rules(input),  // P3 暂不启用：函数在 p3.ts 中完整实现，需要时取消注释
    ...detectOverlap(input),    // 跨 ASIN 词重叠
    ...detectZombies(input),    // 僵尸广告组
  ].sort(byPriority)
}

// lib/rules/sop/p0.ts  —  示例规则
export function applyP0Rules(input: SopInput): ActionItem[] {
  const items: ActionItem[] = []
  const clicksThreshold = input.config("P0_A_clicks_threshold")  // 默认 15
  const spendThreshold  = input.config("P0_B_spend_threshold")   // 默认 20

  for (const row of input.searchTerms) {
    // P0-A: 高点击 0 转化
    if (row.clicks >= clicksThreshold && row.ad_orders === 0) {
      items.push({
        priority: "P0", rule: "P0-A", asin: row.asin,
        term: row.search_term, match: row.match_type,
        clicks: row.clicks, orders: 0, spend: row.spend, acos: null,
        action: "暂停 + 否定",
        reason: `${row.clicks} 次点击，0 成交，花费 $${row.spend.toFixed(1)}`,
      })
    }
    // P0-B: 高花费 0 转化
    if (row.spend > spendThreshold && row.ad_orders === 0) {
      items.push({ priority: "P0", rule: "P0-B", ... })
    }
  }
  return items
}
```

## 四、数据新鲜度判断

Context 面板中文件卡片的颜色状态由以下逻辑决定：

```ts
// lib/config.ts 中定义新鲜度阈值（天）
const FRESHNESS = {
  product:         { fresh: 1, ok: 2 },   // 每日上传，>2 天变 stale
  keyword_monitor: { fresh: 2, ok: 7 },
  search_terms:    { fresh: 5, ok: 14 },
  campaign_3m:     { fresh: 10, ok: 45 },
  us_campaign_30d: { fresh: 5, ok: 10 },
  placement_us_30d:{ fresh: 5, ok: 10 },
  inventory:       { fresh: 3, ok: 7  },
  cost_mgmt:       { fresh: 30, ok: 60 },
  aba_search:      { fresh: 30, ok: 90 },
}

function getFreshness(fileType: string, uploadDate: Date): "fresh" | "ok" | "stale" {
  const daysAgo = (Date.now() - uploadDate.getTime()) / 86400000
  const thresholds = FRESHNESS[fileType]
  if (daysAgo <= thresholds.fresh) return "fresh"
  if (daysAgo <= thresholds.ok)    return "ok"
  return "stale"
}
```
