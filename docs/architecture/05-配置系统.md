# 07 — 配置参数

[← 返回索引](../README.md)

> **目的**：将所有业务阈值从功能逻辑中剥离，集中管理。修改阈值只需改本文件，不需要修改功能模块代码。  
> **层次**：全局默认 → 品类覆盖 → 产品阶段覆盖（优先级从低到高）。  
> **使用方**：广告优化（04）、每日告警（03）、库存看板（05）、品类视图（02）。

---

## 零、品类-ASIN 映射（系统结构基础）

> 这是系统启动的前提配置。所有品类视图的数据范围、内部竞争检测、品类基准匹配，都依赖此映射。

```json
{
  "mattress": {
    "display_name": "床垫",
    "asins": ["B0GD7BF2TZ", "B0GD7K1TC9"]
  },
  "pump": {
    "display_name": "充气泵",
    "asins": ["B0FJKNRNZK"]
  },
  "scooter": {
    "display_name": "电动滑板车",
    "asins": []
  }
}
```

**添加新品类**：在此 JSON 中新增一个 key，填入 ASIN 列表。系统在下次启动时自动识别新品类，无需修改功能代码。

**ASIN 归属规则**：
- 一个 ASIN 只能属于一个品类
- 未在映射中出现的 ASIN，在品类视图中不可见，但 Chat 仍可查询
- 产品报表中的「产品线」字段可作为初始填写参考（当前床垫品类已有值，其余品类需手动填写）

## 参数层次结构

```
CONFIG
├── category_map/        ← 品类-ASIN 映射（系统结构基础，见上）
│
├── global/              ← 全局默认值（覆盖所有品类、所有阶段）
│   ├── sop/             ← 广告 SOP 阈值（P0–P3）
│   ├── alert/           ← 每日告警阈值
│   └── inventory/       ← 库存健康阈值
│
├── category/            ← 品类覆盖（覆盖该品类内所有产品阶段）
│   ├── mattress/        ← 床垫品类
│   ├── scooter/         ← 滑板车品类
│   └── pump/            ← 充气泵品类
│
└── stage/               ← 产品阶段覆盖（按阶段调整告警灵敏度）
    ├── launch/          ← 新品期（< 90天）
    ├── growth/          ← 成长期（90–365天）
    └── mature/          ← 成熟期（> 365天）
```

**覆盖规则**：`stage` > `category` > `global`。若某参数在 category 或 stage 中未定义，自动回退到 global 值。

## 二、每日告警阈值（03 模块）

### 销售指标告警

> **环比基准**：D-1 vs D-2（昨日 vs 前日）。黄/红颜色由下跌幅度 + 持续天数共同决定：单日下跌超阈值 → 黄，连续2日均超阈值 → 红。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `alert_drop_threshold` | 0.20 | 单日环比下跌 > 20% → 黄色（适用 GMV / Sessions / 订单量） |
| `alert_drop_consecutive_days` | 2 | 连续 N 日均下跌超 threshold → 升级红色 |
| `alert_return_rate_yellow` | 0.05 | 退货率 > 5% → 黄色 |
| `alert_return_rate_red` | 0.08 | 退货率 > 8% → 红色 |

### 广告预算告警

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `alert_budget_utilization_low` | 0.70 | 预算利用率 < 70% → 黄色（预算浪费） |
| `alert_budget_utilization_high` | 1.00 | 花费 > 预算 100% → 红色（超预算） |

### ACoS 告警（按产品阶段）

| 阶段 | 黄色阈值（警戒起点） | 红色阈值 | 说明 |
|------|-------------------|---------|------|
| `acos_launch` | 0.50 | 0.70 | 黄色：50–70%；红色：> 70% |
| `acos_growth` | 0.45 | 0.65 | 黄色：45–65%（45–55% 警戒，55–65% 灰色地带）；红色：> 65% |
| `acos_mature` | 0.35 | 0.55 | 黄色：35–55%（35–45% 警戒，45–55% 灰色地带）；红色：> 55% |

### 评分告警（来源：关键词监控报表）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `rating_red` | 3.8 | 评分 < 3.8 → 红色告警（差评风险高，建议暂停广告曝光并跟进 Vine 计划） |
| `rating_yellow` | 4.0 | 评分 < 4.0 → 黄色告警（进入警戒区，主动跟进买家反馈） |

> **数据依赖**：评分数据来自 `keyword_monitor` 报表，仅当该报表已上传时才触发。  
> **BSR 排名**：记录在 ContextFile 中，通过 Chat 工具 `get_file_data("keyword_monitor")` 查询，不触发告警。

## 六、参数加载顺序（伪代码）

```
function getParam(key, category?, stage?):
  if stage and CONFIG.stage[stage][key] exists:
    return CONFIG.stage[stage][key]       // 最高优先级
  if category and CONFIG.category[category][key] exists:
    return CONFIG.category[category][key]  // 品类覆盖
  return CONFIG.global[key]               // 全局默认值（兜底）
```

**示例**：床垫成熟期 ACOS 告警阈值
```
getParam("acos_red", category="mattress", stage="mature")
→ stage.mature 未定义 acos_red
→ category.mattress.acos_red = 0.60  ✅ 使用品类值
```

## 七、新增品类 / ASIN 操作步骤

> **MVP 阶段**：以下操作由开发人员通过 `prisma/seed.ts` 完成，后续版本将提供管理界面。

新增品类或 ASIN 需联动更新以下 4 处，缺少任何一处将导致该品类/ASIN 在对应模块中不可见或计算错误：

### 新增品类（含其下的 ASIN）

```
1. 编辑 prisma/seed.ts
   → CategoryMap：新增一条 { categoryKey, displayName, asins: [...] }
   → AsinConfig：为每个 ASIN 新增一条 { asin, categoryKey, stage, displayName }

2. 运行 seed 更新数据库
   npx prisma db seed

3. 本文件（07-配置参数.md）
   → 零节 category_map JSON：新增品类 key + display_name + asins
   → 四节品类专属基准：新增该品类的 CTR / ACoS / CVR / CPC 基准参数

4. 重新上传该品类对应的报表文件（触发规则引擎写入 Alert / AdAction 表）
```

### 向已有品类新增 ASIN

```
1. 编辑 prisma/seed.ts
   → CategoryMap：在对应品类的 asins 数组中添加新 ASIN
   → AsinConfig：新增一条该 ASIN 的配置记录（指定 stage）

2. npx prisma db seed

3. 本文件 零节 category_map JSON：在对应品类的 asins 中添加新 ASIN
   （不需要修改品类基准参数，除非该 ASIN 的品类阈值与现有设置有显著差异）

4. 重新上传报表（同上）
```

### 修改 ASIN 产品阶段

```
1. 编辑 prisma/seed.ts 中 AsinConfig 的 stage 字段
2. npx prisma db seed
3. 重新上传产品报表触发告警重算
（AsinConfig 无需在本文件中单独维护，seed.ts 是唯一来源）
```
