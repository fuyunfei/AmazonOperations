# 04 — 广告优化行动清单

[← 返回索引](./index.md)

> **数据范围**：ASIN 级（P0/P1/P2 操作）+ 品类级（内部竞争检测）。  
> **触发时机**：上传搜索词重构报表或广告活动重构报表后自动运行。  
> **目标**：将《运营指导手册-广告操作SOP》自动转化为可执行的操作清单，按优先级排序。

---

## 输入报表

> **重要：三份广告报表用途不同，需分别使用**

| 报表 | 来源系统 | 上传频率 | 导出配置 | 主要用途 |
|------|---------|---------|---------|---------|
| 搜索词重构（**ASIN视图**，近30天，US） | Nordhive | 每周 | 站点=US，视图=ASIN视图，时间=近30天 | P0/P1/P2 SOP 规则判断 |
| 广告活动重构（**ASIN视图**，近3个月，ALL） | Nordhive | 每月 | 站点=ALL，视图=ASIN视图，时间=近3个月 | 僵尸广告组识别；长周期 ACOS 评估 |
| US广告活动（**广告活动视图**，近30天，US） | Nordhive | 每周 | 站点=US，视图=广告活动，时间=近30天 | 广告位表现分析；每日预算字段 |
| 广告位报表（US，近30天） | Nordhive | 每周 | 站点=US，时间=近30天 | P1-B CTR 过低时：广告位竞价调整分析 |
| 品类映射配置 | 系统内部 | 一次性 | — | 跨 ASIN 词重叠检测 |

> **时间窗口说明**：实际报表覆盖约30天（例：2026-03-08 至 2026-04-06），**不是7天**。P0规则阈值（clicks ≥ 15，spend > $20）适用于30天累计数据，需在阈值说明中注明统计周期。

## 处理的数据字段

### 来自搜索词重构（对应报表字段名）

```
asin            — ASIN（产品编号）
search_term     — 搜索词（实际触发广告的用户搜索词）
match_type      — 匹配类型（精确 / 词组 / 广泛）
impressions     — 曝光量/展示次数
clicks          — 点击量
spend           — 广告花费（$）
ad_orders       — 广告订单量
ad_sales        — 广告销售额（$）
ctr             — CTR（报表已计算，可直接使用）
cpc             — CPC
acos            — ACoS（报表已计算，可直接使用）
conversion_rate — 转化率（= 广告订单量 ÷ 点击量，报表字段「转化率」）
campaign        — 广告活动名称（用于关联广告活动报表）
placement_type  — 投放类型（SP手动 / SP自动 / SB / SD）
```

### 来自广告活动重构

```
campaign_name   — 广告活动（对应报表字段「广告活动」）
asin            — 所属 ASIN
match_type      — 投放类型（SP手动 / SP自动 / SB / SD）
total_spend     — 广告花费
conversions     — 广告订单量
running_days    = 今日 - 开始日期（报表字段「开始日期」，系统计算）
```

### 来自US广告活动报表（广告位竞价与预算管理）

```
campaign_name      — 广告活动
daily_budget       — 每日预算（$，报表字段「每日预算」）
bidding_strategy   — 竞价策略（固定竞价 / 动态竞价-降低 / 动态竞价-提高和降低）
placement_strategy — 广告位竞价策略（如：搜索结果顶部+X%）
```

### 来自广告位报表（用于 P1-B 广告位分析）

```
placement     — 广告位（搜索结果顶部（首页） / 商品页面 / 其他搜索结果）
campaign_name — 广告活动
impressions   — 曝光量
clicks        — 点击量
ctr           — CTR
acos          — ACoS
spend         — 广告花费
```

## 计算逻辑

### 第一步：比率指标（直接使用报表已计算字段）

```
报表已提供：ACoS、CTR、CPC、转化率（= 广告订单量 ÷ 点击量）
系统需额外计算：
  是否超预算 = 广告花费 > 每日预算（需与US广告活动报表JOIN）
  运行天数   = 当前日期 - 开始日期
```

### 第二步：逐条应用 SOP 规则（ASIN 级别）

> **统计周期**：所有规则基于30天累计数据（报表时间窗口约30天）

规则按优先级分级，触发后生成操作项：

**P0 — 立即止血**

```
规则 P0-A：clicks ≥ 15 且 ad_orders = 0  （30天累计）
  → 建议：暂停该关键词；在父广泛组添加精确否定
  → 原因：高点击0转化，持续消耗预算无回报

规则 P0-B：spend > $20 且 ad_orders = 0  （30天累计）
  → 建议：精确否定该搜索词
  → 原因：无效词花费超阈值

规则 P0-C：广告活动 ACoS > 80% 且 日均花费 > 每日预算
  → 建议：暂停该广泛组，降低主词出价10–15%
  → 数据来源：搜索词重构或广告活动重构（ACoS）+ US广告活动报表（每日预算）
  → 计算：日均花费 = 活动总花费 ÷ 报表天数；与 daily_budget 对比
  → 原因：超预算且严重亏损
  → 限制：需 JOIN US广告活动报表获取每日预算；未上传该报表时此规则不触发
```

**P1 — 今日内优化**

```
规则 P1-A：ACoS 在 80–114% 且 clicks ≥ 30  （30天累计）
  → 建议：出价降低30–40%
  → 原因：有一定成交但 ACOS 严重超标，降价改善效率

规则 P1-B：impressions ≥ 500 且 CTR < 0.2%
  → 建议：排查主图竞争力、价格是否高于竞品 > 15%
  → 追加：从广告位报表查询该广告活动各广告位的 CTR 对比
          若「搜索结果顶部」CTR 正常但「其他搜索结果」CTR 低
          → 建议：调整广告位竞价策略，降低非顶部广告位的竞价调整比例
  → 原因：展示量充足但点击率过低，问题在 listing 本身

规则 P1-C：精确词累计 impressions < 500 且运行 ≥ 7天
  → 建议：出价提升20–30%
  → 原因：曝光量不足，出价可能低于竞争门槛
```

**P2 — 本周内优化**

```
规则 P2-A：ACOS ≤ 35% 且 CVR ≥ 4% 且 clicks ≥ 30  （30天累计）
  → 建议：出价提升15–20%
  → 原因：高效词，扩大流量获取

规则 P2-B：同一 ASIN 内，同一搜索词同时出现在广泛匹配和精确匹配广告组
  → 建议：精确匹配组出价设为高于广泛匹配组5–10%；广泛匹配组对该词添加精确否定
  → 数据来源：搜索词重构（按 ASIN + search_term 分组，检查 match_type 是否同时含广泛和精确）
  → 原因：同 ASIN 内广泛与精确同时投放同一词会导致自竞价，推高 CPC

规则 P2-C：ASIN 定投广告组 ACoS > 70% 且 运行 > 14天
  → 建议：①筛选转化率高的竞品 ASIN 保留 ②删除 clicks > 10 且 0 成交的竞品 ASIN ③整组出价降15%
  → 数据来源：广告活动重构（通过活动名识别 ASIN 定投组，名称含「ASIN」「PAT」「定投」）
              + US广告活动报表（开始日期，用于计算运行天数）
  → 原因：ASIN 定投长期高 ACOS 说明目标竞品选择或出价需要调整
  → 限制：运行天数需 JOIN US广告活动报表；无法获取开始日期时不触发

规则 P2-D：词组匹配 30天总曝光 < 3000（即日均 < 100）
  → 建议：出价提升25–35%
  → 数据来源：搜索词重构（筛选 match_type = 词组匹配）
  → 原因：词组匹配曝光不足，出价低于入场门槛，需尽快获取数据
  → 说明：报表覆盖约30天，用30天总曝光判断，不依赖运行天数字段

规则 P2-E：广告活动 ACoS > 70% 且 运行 > 14天  （广告活动重构维度）
  → 建议：筛查广告组关键词，低效词降出价15%
  → 数据来源：广告活动重构（ACoS）+ US广告活动报表（开始日期，用于计算运行天数）
  → 原因：持续高 ACOS 但运行时间足够长，说明词或出价结构需要调整
  → 限制：运行天数需 JOIN US广告活动报表；无法获取开始日期时不触发
```

**P3 — 本月内优化**

```
规则 P3-A：广泛/词组词 clicks ≥ 20 且 CVR ≥ 3%  （30天累计）
  → 建议：将该搜索词添加到精确匹配广告组，初始出价 = 当前出价 × 1.2
  → 原因：广泛/词组词已验证转化，提炼精确词可提高效率、降低浪费

规则 P3-B：单个广告组内去重搜索词数量 > 50个
  → 建议：按效率分层——高效词（CVR ≥ 3%）保留，低效词（clicks ≥ 15 且 CVR = 0）移至否定列表或独立测试组
  → 数据来源：搜索词重构（按「广告组」字段分组，统计去重搜索词数量）
  → 原因：广告组过大导致预算分散，难以针对性优化
  → 说明：此处计数的是搜索词重构报表中按广告组分组后的去重搜索词数，
          不等于广告后台设置的关键词投放数量（一个投放关键词可触发多个搜索词）

规则 P3-C：历史同期（YoY）GMV 增长 > 20%（季节性检测）
  → 建议：提前2–3周将主词出价提升15–20%，日预算增加30–50%
  → 数据来源：产品报表时序（同期对比）
  → 原因：季节性旺季流量增大，提前布局避免预算不足错失流量
  → 备注：需产品报表有历史同期数据（≥13个月时序）方可触发

规则 P3-D：品牌词防御  ⚠️ 依赖 ABA 数据（当前暂未启用）
  → 触发条件：ABA 竞品点击份额 > 5%
  → 建议：开启品牌词精确匹配广告，出价 $0.5–$1.0
  → 数据来源：ABA 搜索词对比报表（aba_search_compare）
  → 原因：竞品抢占品牌词流量超阈值，需主动防御
  → 备注：ABA 功能暂未启用，规则已定义但不触发
```

### 第三步：跨 ASIN 词重叠检测（品类级别，生成 P2 内部竞争项）

```
遍历本品类内所有 ASIN 的搜索词报告
对每个 (search_term, match_type) 组合：
  若同一词 × 同一投放类型 × 出现在 ≥ 2 个不同 ASIN 的广告活动中
    → 标记为「品类内部竞争」
    → 归类为 P2，建议：在低效 ASIN 的广告组中否定该词

示例：
  床垫品类中，B0GD7BF2TZ 和 B0GD7K1TC9 都以「精确」投放「sofa mattress」
  → P2：内部竞争，建议在 Full 款中否定该词，让流量集中到 Queen 款
```

### 第四步：僵尸广告组识别（广告活动重构维度）

```
条件：total_spend > 0 且 conversions = 0 且 running_days > 30
  → 标记为「僵尸广告组」
  → 建议：暂停该广告活动，预算转移到高效组
```

## 优先级说明

| 优先级 | 标签 | 处理时限 | 规则数 |
|--------|------|---------|--------|
| P0 | 止血 | 立即（当日）| P0-A / P0-B / P0-C |
| P1 | 优化 | 今日内 | P1-A / P1-B / P1-C |
| P2 | 扩量 / 调整 | 本周内 | P2-A / P2-D / P2-E |
| P3 | 结构优化 | 本月内 | P3-A / P3-B / P3-C / P3-D（暂停） |

---

## 注意事项

- **搜索词重构报表**：Nordhive 导出时选 All 站点、最近30天；选 ASIN 视图
- **广告活动重构报表**：Nordhive 导出时选 ALL 广告类型、最近3个月
- **品类内部竞争检测**必须依赖品类映射配置，否则无法识别哪些 ASIN 同属一个品类
- **P3-C 季节性规则**需产品报表积累 ≥ 13个月时序数据才可触发
- **P3-D 品牌词防御**依赖 ABA 数据，当前暂未启用，规则保留待后续激活
- SOP 阈值来源：《运营指导手册-广告操作SOP》P0–P3 规则；所有数值阈值均通过配置参数管理，见 [→ 07-配置参数.md](./07-配置参数.md)
