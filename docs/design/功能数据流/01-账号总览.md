# 01 — 账号总览页

[← 返回索引](./index.md)

> **数据范围**：全账号所有 ASIN 聚合，不区分品类。  
> **刷新时机**：每次上传任意报表后自动重算。

---

## 功能说明

账号总览是系统的根层级页面，展示：

1. **账号级 KPI 汇总** — 跨所有品类的总量指标
2. **数据源新鲜度面板** — 各报表的上次上传时间和状态
3. **品类健康状态列表** — 每个品类的告警摘要入口

---

## 输入报表

| 报表 | 来源系统 | 上传频率 | 用途 |
|------|---------|---------|------|
| 产品报表-子ASIN视图（单日快照） | Nordhive | 每日 | GMV、订单量、广告花费、广告销售额 |
| 库存报表 | Nordhive | 按需（有变化时） | 全账号可售库存汇总 |
| 品类映射配置 | 系统初始化时配置 | 一次性 / 变更时 | 决定哪些 ASIN 属于哪个品类 |

---

## 处理的数据字段

### 来自产品报表（所有 ASIN 行求和）

| 字段 | 处理方式 |
|------|---------|
| `gmv` | `SUM(所有ASIN近30天)` → 账号总 GMV |
| `ad_spend` | `SUM(所有ASIN近30天)` → 账号总广告花费 |
| `orders` | `SUM(所有ASIN近30天)` → 账号总订单量 |

### 来自库存报表（所有 SKU 行求和）

| 字段 | 处理方式 |
|------|---------|
| `sellable_qty` | `SUM(所有SKU)` → FBA 总可售库存件数 |

---

## 计算逻辑

```
账号 TACoS = SUM(账号所有ASIN的ad_spend, 近30天)
           ÷ SUM(账号所有ASIN的gmv, 近30天)

账号级 KPI 均为「累加」，不做加权平均
（ACOS、CTR 等比率指标不在账号总览展示，避免误导）

数据源新鲜度判断：
  fresh  → 上传时间距今 ≤ 1天（产品报表）/ ≤ 规定上传周期内
  ok     → 在预期上传周期内
  stale  → 超过预期周期未更新
    产品报表：> 2天未上传 → stale
    搜索词重构：> 14天未上传 → stale
    广告活动重构：> 45天未上传 → stale
    ABA搜索词对比：> 45天未上传 → stale
```

## 输出展示

```
账号总览页展示内容：
  ┌─────────────────────────────────────┐
  │  账号总 GMV      账号总广告花费      │
  │  账号 TACoS      FBA 总可售库存      │
  └─────────────────────────────────────┘

  数据源新鲜度列表：
    产品报表      新鲜  ← 今日已上传
    关键词监控    新鲜
    库存报表      正常  ← 昨日上传
    搜索词重构    正常  ← 4天前
    广告活动重构  正常  ← 12天前
    ABA搜索词对比 过期  ← 32天前（超期）

  品类健康状态：
    床垫    2红 1黄  → 点击进入品类视图
    充气泵  2红      → 点击进入品类视图
    滑板车  1黄      → 点击进入品类视图
```

---

## 不在账号总览展示的内容

- 各 ASIN 的 ACOS、CTR、CVR 等比率指标（需在 ASIN 级别查看）
- 广告优化行动清单（ASIN 级别功能）
- 库存补货建议（SKU 级别功能）
