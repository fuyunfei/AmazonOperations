## 数据流总览

```
[Nordhive 导出 / Seller Central 导出]
         │
         ▼
[摄入层]  用户上传 .xlsx → 系统解析 → 提取原始计数字段 → 校验完整性
         │
         ▼
[存储层]  时间序列表（ASIN × 日期）
         快照表（库存 / 搜索词 / 广告活动 / 成本 — 保留最新版本）
         │
         ▼
[计算层]  按需聚合任意时间窗口 → 计算派生比率指标 → 与阈值对比
         │
         ▼
[输出层]  告警触发 / 广告行动清单 / 库存预警 / Chat 上下文注入
```

## 已知数据缺口

| 手册要求 | 字段来源 | 状态 | 处理方案 |
|---------|---------|------|---------|
| Buy Box 占比（每日必看） | Seller Central 业务报告 | ❌ Nordhive产品报表无此字段 | MVP用`OCR`（订单转化率）作间接替代；精确数据需从SC额外下载Business Report |
| ABA 搜索词（每月必看） | `US_ABA_Search_Compare_*.xlsx` | ⚠️ 有文件但无独立功能模块 | 作为Chat工具调用数据源（`get_file_data('aba_search_compare')`） |
| 品类映射 | 产品报表/库存报表`产品线`字段 | ⚠️ 字段存在但填写不完整（仅床垫2个ASIN有值） | 系统需品类映射配置页面，以`产品线`字段预填，其余用户手动分配 |

## 存储字段原则

- **存原始计数，不存比率**：`ad_spend`、`ad_sales`、`clicks`、`impressions`、`orders`、`gmv`
- **比率由系统计算**：`ACOS = ad_spend / ad_sales`、`CTR = clicks / impressions`、`CVR = orders / clicks`
- **时间序列**：产品报表按 `asin × date` 逐日累积，不覆盖
- **快照表**：搜索词、广告活动、库存、成本按最新版本覆盖存储，带 `snapshot_date` 字段
