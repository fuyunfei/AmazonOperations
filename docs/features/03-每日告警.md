# 03 — 每日告警看板

[← 返回索引](../README.md)

> **数据范围**：单 ASIN。  
> **触发时机**：每日上传产品报表后自动运行。  
> **目标**：用红/黄/绿三色标记每个 ASIN 当前的健康状态，并给出操作建议。

---

## 输入报表

| 报表 | 来源系统 | 上传频率 | 提供字段 |
|------|---------|---------|---------|
| 产品报表-子ASIN视图（**昨日单日快照**） | Nordhive | 每日 | GMV、订单量、退货量、广告花费、广告销售额、点击量、曝光量、Sessions |
| 关键词监控（最新快照） | Nordhive | 每日 | 评分、评论数、BSR大类/小类排名、关键词自然排名、SP广告排名、综合排名 |
| 库存报表（最新） | Nordhive | 按需 | 可售库存量（用于计算可售天数） |
| 产品报表历史时序（近7天已累积） | 系统内部 | 自动积累 | 日GMV、日订单量（用于计算环比和周均） |

> **注意**：产品报表每日只上传「单日快照」（昨天一天的数据），系统自动从历史时序计算多日聚合。

---

## 处理的数据字段

### 产品报表原始字段（存储，不直接展示）

```
asin          — 产品编号
date          — 报表日期
gmv           — 当日销售总额
orders        — 当日订单量
returns       — 当日退货量
ad_spend      — 当日广告花费
ad_sales      — 当日广告销售额
clicks        — 当日广告点击量
impressions   — 当日广告曝光量
sessions      — 当日页面访问量（对应报表字段「总流量」）
ocr           — 订单转化率（对应报表字段「OCR」，作为 Buy Box 状态的间接替代）
```

> **Buy Box 说明**：Nordhive 产品报表**不含 Buy Box 占比字段**。`OCR`（订单转化率 = 订单量 / 总流量）是间接替代指标，OCR 显著低于品类基准通常是 Buy Box 丢失的信号。精确 Buy Box 数据需从 Seller Central 业务报告（Detail Page Sales and Traffic by Child Item）单独下载。

### 关键词监控原始字段

```
asin              — 产品编号
keyword           — 监控的关键词
natural_rank      — 自然排名（含变化方向，如 "26↓(2)"）
sp_rank           — SP广告排名
overall_rank      — 综合排名
rating            — 当前评分
review_count      — 评论数量
bsr_main          — BSR大类排名
bsr_sub           — BSR小类排名
updated_at        — 数据更新时间
```

### 库存报表原始字段

```
asin              — 产品编号
sku               — 子SKU编号
sellable_qty      — 当前可售库存量
snapshot_date     — 快照日期
```

## 计算逻辑

### 第一步：从原始计数计算比率指标（ASIN 级别）

> **注意：CVR 在产品报表中有两个不同定义，必须区分**

```
广告指标（来自产品报表广告字段）：
  ACOS      = 广告花费 ÷ 广告销售额
  ROAS      = 广告销售额 ÷ 广告花费
  广告CTR   = 广告点击量 ÷ 广告曝光量
  广告CVR   = 广告订单量 ÷ 广告点击量        ← 报表字段「广告订单量 ÷ 广告点击量」
  广告花费利用率 = 广告花费 ÷ 每日预算        ← 需与US广告活动报表JOIN取「每日预算」

页面指标（来自产品报表流量字段）：
  OCR（页面转化率）= 订单量 ÷ 总流量         ← 报表字段「OCR」，≠ 广告CVR
  退货率    = 退款数量 ÷ 订单量               ← 报表字段「退款率」

库存指标：
  可售天数  = 可售库存 ÷ (SUM(订单量, 近7天) ÷ 7)
```

### 第二步：环比计算（依赖时间序列历史）

```
环比基准：昨日（D-1）vs 前日（D-2），与手册"每日检查"节奏一致。

GMV环比         = 销售额(D-1) ÷ 销售额(D-2) - 1
订单环比        = 订单量(D-1) ÷ 订单量(D-2) - 1
Sessions环比    = 总流量(D-1) ÷ 总流量(D-2) - 1    ← 手册要求，流量异常是排查优先项

告警升级规则（连续天数维度）：
  单日下跌 > 20% → 黄色告警
  连续 2 日均下跌 > 20% → 自动升级为红色告警
  （判断连续2日需读取 D-1、D-2、D-3 三天数据，系统至少需要积累 3 天产品报表）
```

### 第三步：与阈值对比，打红/黄/绿标

**产品阶段来源**：报表中无此字段，需在 `AsinConfig` 中由用户手动设置（新品期 / 成长期 / 成熟期）。

| 指标 | 绿色（正常） | 黄色（关注） | 红色（告警） | 依赖字段 |
|------|------------|------------|------------|---------|
| ACOS（成熟期） | ≤ 35% | 35–55%（注1） | > 55% | 广告花费 / 广告销售额 |
| ACOS（成长期） | ≤ 45% | 45–65%（注1） | > 65% | 同上 |
| ACOS（新品期） | ≤ 50% | 50–70% | > 70% | 同上 |
| 退货率 | < 5% | 5–8% | > 8% | 退款率（退款数量/订单量） |
| 可售天数 | ≥ 45天 | 30–45天 | < 30天 | 可售库存 + 近7天订单时序 |
| 广告CTR | ≥ 0.3% | 0.2–0.3% | < 0.2% | 广告点击量 / 广告曝光量 |
| OCR（页面转化率） | ≥ 10% | 8–10% | < 8% | OCR 字段（订单量/总流量） |
| 评分 | ≥ 4.0 | 3.8–4.0 | < 3.8 | 关键词监控报表：评分 |
| GMV环比 | 单日下跌 ≤ 20% | 单日下跌 > 20% | 连续2日下跌 > 20% | 销售额时序（D-1/D-2/D-3） |
| 订单环比 | 单日下跌 ≤ 20% | 单日下跌 > 20% | 连续2日下跌 > 20% | 订单量时序（D-1/D-2/D-3） |
| Sessions环比 | 单日下跌 ≤ 20% | 单日下跌 > 20% | 连续2日下跌 > 20% | 总流量时序（D-1/D-2/D-3） |
| 广告花费利用率 | 70–100% | < 70% | > 100%（超预算） | 广告花费 / 每日预算 |

> **注1 — ACOS 黄色区间内的灰色地带**：黄色告警覆盖整个"35–55%"或"45–65%"范围，但内部有两段含义不同：
> - 成熟期 35–45% / 成长期 45–55%：**警戒区**，效率下滑，需关注优化
> - 成熟期 45–55% / 成长期 55–65%：**灰色地带**，广告可能亏钱但尚未达到红线，需主动做搜索词优化和出价调整
>
> 两段均触发黄色告警，但建议文案应有所区分（实现时可用 sub-threshold 区分严重程度）。

> **广告花费利用率**依赖`US广告活动报表`中的`每日预算`字段，需与产品报表按广告活动名称JOIN。若未上传该报表，此告警项不触发。

### 第四步：匹配建议操作（来自手册 SOP）

每条告警触发时，系统从操作手册中查找对应处理建议并附加到告警卡片。

```
示例：
  指标：ACOS > 55%（红）
  建议：拉搜索词报告，否定点击≥15且0成交的词；
        检查高花费低转化广告活动，考虑暂停广泛组
  
  指标：Sessions环比 < -30%（红）
  建议：检查是否丢失Buy Box（OCR是否同步下降）；
        检查关键词自然排名是否下滑；确认Listing状态正常
```

## 数据刷新依赖

```
必须已上传：
  ✅ 今日产品报表（昨日单日快照）  → 计算 ACOS/CTR/CVR/退货率/环比
  ✅ 关键词监控报表               → 计算评分/排名相关告警
  ✅ 库存报表                     → 计算可售天数

系统内部依赖：
  ✅ 近3天历史时序（D-1/D-2/D-3）  → 计算日环比 + 连续天数升级判断（最少需3天才能触发红色）
  ✅ 近7天历史时序                 → 计算可售天数日均销量（日均销量 = SUM(近7天订单) / 7）
  （历史时序由每日产品报表累积；首次上传只有1天时，环比类告警不触发）
```
