# 05 — 库存健康看板

[← 返回索引](../README.md)

> **数据范围**：ASIN 下展开至 SKU 级别。  
> **触发时机**：上传库存报表后自动运行。  
> **目标**：识别断货风险、长期仓储费预警、滞销库存，并给出补货建议量。

---

## 输入报表

| 报表 | 来源系统 | 上传频率 | 提供字段 |
|------|---------|---------|---------|
| 库存报表（最新快照） | Nordhive | 按需（有变化时） | SKU、可售库存、在途库存各阶段、库龄分布、月度仓储费、长期仓储费 |
| 产品报表历史时序（近30天已累积） | 系统内部 | 自动积累 | 各 SKU/ASIN 日均订单量 |
| 成本管理 | 用户手动更新 | 变更时 | SKU货值、FBA费用（用于计算积压成本） |

---

## 处理的数据字段

### 来自库存报表

```
asin                 — 产品编号
sku                  — SKU 编号（如 Queen-4IN-US）
site                 — 站点（US / CA / MX）
sellable_qty         — 当前可售库存件数
inbound_working      — 在途库存-已创建
inbound_shipped      — 在途库存-已发货
inbound_receiving    — 在途库存-接收中
aging_0_30           — 0-30天库存
aging_31_60          — 31-60天库存
aging_61_90          — 61-90天库存
aging_91_180         — 91-180天库存
aging_181_330        — 181-330天库存（含181-210/211-240/.../301-330各子段，长仓费关键区间）
aging_331_365        — 331-365天库存
aging_365_plus       — 大于365天库存
monthly_storage_fee  — 月度仓储费（$）
long_term_fee_charged — 长期仓储费 已收取（$）
snapshot_date        — 报表快照日期

> **库龄字段说明**：报表同时提供两套库龄数据：
> - **汇总版**（用于健康判断）：0-30 / 31-60 / 61-90 / 91-180 / 181-330 / 331-365 / 365+
> - **细分版**（用于精确长仓费计算）：181-210 / 211-240 / 241-270 / 271-300 / 301-330 / 331-365 / 365+，每段附带仓储费字段
```

### 来自产品报表历史时序（系统计算）

```
orders_last30        — 该 SKU 近30天总订单量（从时序表聚合）
daily_avg_sales      = orders_last30 ÷ 30
```

### 来自成本管理

```
sku                  — SKU 编号
unit_cost            — 单件货值（$）
fba_fee              — FBA 配送费（$）
```

---

## 计算逻辑

### 第一步：核心健康指标（每个 SKU 独立计算）

```
日均销量   = orders_last30 ÷ 30

可售天数   = sellable_qty ÷ 日均销量
            （日均销量 = 0 时，标记为「零销量」）

在途库存   = inbound_working + inbound_shipped + inbound_receiving

补货建议量 = 日均销量 × (海运天数 + 30天安全库存) - sellable_qty - 在途库存
            （结果 ≤ 0 → 无需补货）
            默认海运天数：21天（可在成本管理中配置）

积压成本   = sellable_qty × (unit_cost + fba_fee)
```

### 第二步：断货风险分级

```
可售天数 < 30天                         → 🔴 红色，立即补货
可售天数 30–45天                        → 🟡 黄色，准备补货计划
可售天数 < 45天 且 在途库存 = 0         → 🟡 黄色加重预警（无在途缓冲）
可售天数 ≥ 45天                         → 🟢 正常
日均销量 = 0（近30天无销量）             → ⚪ 零销量，特殊标记
```

### 第三步：长期仓储费风险

```
使用汇总版库龄字段判断：
  aging_181_330 > 0                      → 🟡 黄色，长仓费风险（将在下次收费日触发）
  aging_331_365 > 0 或 aging_365_plus > 0 → 🔴 红色，高额长仓费（每月15日自动扣费）

精确费用展示（使用细分版字段）：
  各子段（181-210/211-240/241-270/271-300/301-330/331-365/365+）
  报表直接提供各段的「仓储费」字段，可直接累加展示预计月度长仓费
```

> 亚马逊每月15日对库龄 > 181天的库存收取长期仓储费（约 $6.90/立方英尺）。

### 第四步：滞销识别

```
条件：可售天数 > 90天 且 近30天销量 < 5单
  → 标记为「滞销」
  → 建议清库路径：
      1. 降价促销（价格降至竞品前30%）
      2. 站内 LD/BD 促销清库
      3. 移仓至第三方仓储
      4. 申请销毁（成本低于长仓费时）

积压成本计算：
  滞销积压成本 = sellable_qty × (unit_cost + fba_fee)
  展示以帮助决策是否值得清库
```

## 注意事项

- **库存报表**：Nordhive 导出时选最新快照，包含库龄分布字段
- **补货建议量计算**依赖近30天历史时序；首次使用时若历史不足30天，日均销量会偏低，建议参考时谨慎使用
- **多站点**：CA / MX 站点的库存单独计算可售天数，不与 US 合并
- 阈值来源：《运营指导手册-每日必看》库存相关规则
