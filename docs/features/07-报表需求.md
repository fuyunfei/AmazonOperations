## 二、各报表所需列字段

### 2.1 product（产品报表 - ASIN 视图）

**格式**：Nordhive 标准格式（Row1 筛选标签，Row2 筛选值，Row4 列头，Row6+ 数据）  
**解析器**：`parseAsinReport.ts`  
**必需列**：ASIN、SKU、站点、日期/时间范围  
　　→ Sessions、订单量、GMV、广告花费、广告销售额、广告订单量、曝光量、点击量、退货量  
**当前状态**：✅ 实际报表完整，另有 5 个模拟日期文件（20260406-20260411）

---

### 2.2 search_terms（搜索词重构 - ASIN 视图，30天）

**格式**：Nordhive 标准格式  
**解析器**：`parseSearchTerm.ts`  
**必需列**：

| 列名 | 用途 |
|------|------|
| ASIN | 品类过滤 |
| SKU | 辅助识别 |
| 站点 | 筛选 US |
| 搜索词 | 核心维度 |
| 匹配类型 | 精确/词组/广泛 |
| 竞价 | P0-A 竞价建议 |
| 广告活动 | P2 跨 ASIN 词重叠检测 |
| 投放类型 | SP手动/SP自动区分 |
| 曝光量/展示次数 | CTR 计算 |
| 点击量 | 核心指标 |
| 广告花费 | ACoS 计算（取绝对值） |
| 广告销售额 | RoAS 计算 |
| 广告订单量 | CVR 计算 |
| ACoS | P0-A/P1-A 高ACoS规则 |

**问题**：Nordhive 实际导出为 **SKU 视图**（14列，仅产品标识信息），缺少全部广告表现列。  
**原因**：导出时选择了 SKU 视图而非 ASIN 视图，且未勾选广告指标列。  
**正确导出步骤**：
```
Nordhive → 广告 → 搜索词重构
  ✓ 视图：ASIN 视图（非 SKU 视图）
  ✓ 站点：US
  ✓ 时间范围：近 30 天
  ✓ 勾选列：ASIN、搜索词、匹配类型、竞价、广告活动、曝光量、点击量、广告花费、广告销售额、广告订单量、ACoS
```
**临时方案**：使用 `搜索词重构_US_30d_2026-04-09_simulated.xlsx`（20 条含真实结构的模拟数据）

### 2.6 inventory（库存报表 - 多站点）

**格式**：Nordhive 标准格式  
**解析器**：`parseInventory.ts`  
**必需列**：

| 列名 | 用途 |
|------|------|
| SKU / ASIN / 站点 | 标识 |
| 可售库存 | 核心库存量 |
| 不可售库存 / 预留库存 | 全量盘点 |
| 在途库存-已创建/已发货/接收中 | 补货进度 |
| 库存货值 / 库存成本 | 资金占用 |
| 库销比 | 库存健康 |
| 月度仓储费 / 长期仓储费 已收取 | 费用监控 |
| 0-30/31-60/61-90/91-180/.../大于365天库存 | 库龄分布 |

**问题**：实际导出 11 列，仅含产品标识（ASIN、SKU、商品名称等），缺少全部库存数量列。  
**正确导出步骤**：
```
Nordhive → 库存 → 库存报表
  ✓ 站点：多站点
  ✓ 勾选列：可售库存、不可售库存、预留库存、在途库存各状态、库存货值、月度仓储费、库龄分布（0-30天等7档）
```

## 五、上传优先级建议

首次启动 MVP 时，按以下顺序上传文件（每上传一个报表后，相关功能模块自动激活）：

```
第1步：产品报表（ASIN视图）× 多日期
  → 解锁：KPI看板、告警看板、Overview总览

第2步：搜索词重构_simulated + 广告活动重构_simulated
  → 解锁：广告优化行动清单（SOP P0-P2）

第3步：US广告活动（30天）（真实报表）
  → 解锁：预算利用率告警（ads.ts 规则）

第4步：库存报表_simulated
  → 解锁：库存健康看板、可售天数告警

第5步：成本管理_simulated
  → 解锁：Chat 工具 get_file_data("cost_mgmt") 查询利润数据

第6步：关键词监控（真实报表）
  → 解锁：BSR/评分追踪（Chat 查询）

第7步（可选）：广告位报表（真实报表）
  → 广告位 CTR 分析（P1-B 规则）
```

## 六、正式数据迁移路线

当 Nordhive 报表配置修正后，替换步骤：

1. **重新导出**按第二节各报表的正确步骤导出
2. 上传新报表到系统（`/api/upload` 会自动 upsert 覆盖同类型旧数据）
3. 验证：上传后查看对应功能面板数据是否正常刷新
4. 删除 context/ 目录中对应的 `*_simulated.xlsx` 文件（通过 `/api/files` DELETE 接口）
