Product Definition

AmazonAgent 通过广告决策，缩小 SKU 运营目标与当前表现之间的差距

01 / 产品目标

诊断 Gap，输出广告层面能做什么来缩小它

用户的 SKU 运营计划（OP）:
  目标销量    = 300 件/月
  目标均价    = $45
  目标毛利率  = 20%

当前轨迹:
  当前日销    = 5 件（需要 10 件才能达标）
  当前 ACOS   = 35%
  当前毛利率  = 12%

Gap = OP 目标 − 当前轨迹

广告是缩小 Gap 的杠杆之一。产品不替用户做所有决策，但在广告这根杠杆上，告诉用户：该加力、该收力、还是该换方向。

  • 销量缺口 → 流量不足（加预算/拓词）或转化率低（关键词不匹配）
  • 利润缺口 → ACOS 超盈亏线（砍无效词）或 CPC 过高（调出价）
  • 增长健康 → 自然单占比在上升吗？BSR 在改善吗？
02 / 核心洞察

真正稀缺的不是数据，是判断

Amazon 广告后台有几千行数据。Helium 10、Jungle Scout 给你更多。数据从来不缺。

缺的是一个高级运营的判断力：看到同样的数据，知道什么该停、什么该加、什么再等等。这个判断力在一个人的脑子里 — 他走了，判断力就没了。

高级运营做判断时，脑子里有三样东西：

  1. 业务目标：这个产品要做到月销 300 件、毛利 20%，现在差多少
  2. 经济参数：盈亏平衡 ACOS 是多少，过了这条线就是亏
  3. 阶段意识：新品期 ACOS 200% 可以接受，成熟品不行；新关键词跑 3 天没曝光要查原因，老关键词 14 天没转化要停

Amazon 后台只给你当前状态。没有一个工具把这三样东西和数据放在一起做判断。现有工具对所有产品用同一套规则，因为它们不知道每个产品的目标、经济参数和阶段。

这导致两个可观察的现象：

决策疲劳是没有决策框架的症状。不知道目标和阶段，每一行数据都"可能重要"。有了框架，95% 的数据不需要看 — 没有穿越任何阈值，不需要行动。

跨数据源分析是做判断的前提条件，不是功能卖点。诊断"销量为什么没达标"需要交叉广告数据 + 销售数据 + 经济参数 + BSR 趋势。单一数据源无法诊断。

AmazonAgent 的核心：把高级运营的判断逻辑编码成系统 — 基于用户的业务目标、产品经济参数和所处阶段，从数据中生成决策信号。

人走了，判断力还在。

03 / 用户价值

OP 达成率提升 + 广告浪费减少

  • 缩小 Gap — 诊断"为什么没达标"并给出广告动作：流量不足则拓词加预算，转化率低则排查关键词匹配度，ACOS 超线则砍无效支出
  • 减少浪费 — 识别正在亏钱的广告支出（零成交词、ACOS 超盈亏线的广告组），量化每周损失金额

每条信号都有明确的美元金额。用户的价值感知不是"我用了一个工具"，而是"这条信号帮我发现日销差 5 件是因为广告流量不够"或"这条信号帮我每周省了 $200"。

更深层的价值：不再依赖某一个高级运营的个人经验。判断逻辑在系统里，人员流动不影响决策质量。

04 / 用户与场景

任何需要判断"这笔广告费该不该继续花"的人

不按职位定义用户，按场景定义。

场景典型用户使用方式
自己管广告的个体卖家年销 $10-50 万每周看信号，直接操作
管多个产品的运营主管带 3-10 人团队看信号，分配任务给运营
为广告付钱的老板有运营团队看 P0 信号，拍板大决策
代运营服务商管多个客户账号用信号向客户汇报和建议

共同特征：对广告花费有决策权或问责权，但没有时间逐行看数据。

05 / 决策模型

三个输入，两层阶段，Gap 驱动的信号

5.1 三个输入

决策模型需要三个输入，缺一不可：

1
系统计算
盈亏平衡线
(售价 − 成本 − FBA费 − 佣金) / 售价。过了这条线就是净亏损。
2
用户设定
ACOS 目标
广告效率期望，反映利润率目标。通常低于盈亏线。
3
用户设定
SKU 运营计划
{月销量, 均价, 毛利率} — 广告决策的业务锚点。
OP（业务目标）  → 决定广告需要完成什么
盈亏线（经济约束）→ 决定广告不能超过什么
ACOS 目标（效率期望）→ 决定广告应该做到什么

没有 OP，系统只能说"ACOS 太高"。有了 OP，系统能说"日销差 5 件是因为广告流量不足 — 当前 50 次点击/天 × 10% CVR = 5 单，目标 10 单需要 100 次点击"。

5.2 分析粒度

信号在广告组级别生成，下钻到关键词级别提供证据。广告组是"策略意图一致"的最小单元 — 同一组内关键词共享匹配类型和出价策略。不在关键词级别直接判断（会误杀引流词），不在活动级别（粒度太粗）。

5.3 信号生成：从 Gap 到广告动作

第一步：计算 Gap
  Gap(销量) = 目标月销 − 当前轨迹月销
  Gap(利润) = 目标毛利率 − 当前毛利率
  Gap(效率) = 目标 ACOS − 当前 ACOS

第二步：诊断原因
  销量 Gap → 流量不足？→ 加预算、拓词
           → CVR 低？→ 关键词不匹配（标记但不修 Listing）
  利润 Gap → ACOS 超盈亏线？→ 识别亏损关键词
           → CPC 过高？→ 调出价
  效率 Gap → 零成交高消耗？→ 否词/暂停
           → 高 ROI 未扩量？→ 加预算/升级匹配

第三步：排优先级 ↓
P0 / 今天处理
正在亏钱，或阶段约束被违反。不行动则持续损失。
P1 / 本周处理
有明确 Gap 且有可执行的广告动作，尚不紧急。
P2 / 两周内
优化机会，可以更好但当前不亏。
Good / 可扩大
表现好于目标，确认并寻找扩量空间。

每条信号附带：证据（哪些关键词/广告组）、影响（每周 $ 金额）、趋势（恶化/改善）、动作（具体建议）。

5.4 两层阶段模型

阶段感知分两层：产品阶段决定整体策略，关键词生命周期决定单个关键词的评判标准。

产品阶段 — 用户手动标记

阶段Gap 诊断重心约束P0 触发
new曝光和初单总支出 < 预算上限曝光 < 500/周
early关键词发现和排名单词浪费 < $20零成交 > $20
growth缩小 OP GapBSR 不连续下降ACOS > 盈亏线
mature利润最大化自然占比不降 + BSR 稳定ACOS > 目标

关键词生命周期 — 系统自动计算

阶段时间评判标准信号
发现期Day 1–3有没有曝光？零曝光 → 出价太低
验证期Day 4–7有没有点击？CTR 极低 → 词不匹配
确认期Day 8–14有没有转化？零转化 → 流量质量问题
优化期Day 15+赚不赚钱？标准盈亏评估

两层组合的效果：growth 阶段产品上刚加的新关键词 — 产品用 growth 约束（ACOS > 盈亏线 = P0），但关键词才跑 3 天还在发现期 — 此时不判断 ACOS，只看有没有曝光。避免误杀正在测试的新词。

06 / 边界

输入是广告数据，输出是决策信号，执行靠人

做什么

  • 接收 SKU 运营计划（OP）
  • 读取 Amazon 广告数据
  • 计算盈亏平衡线
  • 诊断 OP 与现状的 Gap
  • 产品阶段 + 关键词周期生成信号
  • 量化每条信号的 $ 影响
  • 决策闭环（确认/拒绝/推迟）
  • 追踪决策效果

不做什么

  • 自动执行（调价、暂停词）
  • 库存管理
  • 自动改价 / Repricing
  • Listing 优化 / 文案生成
  • 账号合规 / 申诉
  • 多平台
  • 站外获客
  • 定义阶段
输入是广告数据，输出是决策信号，执行靠人。
