# Max CPC：广告决策的唯一锚点

## 策略是什么

策略 ≠ 操作步骤。策略回答的是：**在约束条件下，如何分配有限资源，以实现目标。**

| 要素 | 在我们的框架中 |
|------|--------------|
| **目标** | maximize 利润（TACoS↓ + Margin 不被侵蚀） |
| **约束** | 实际 CPC ≤ Max CPC |
| **资源分配** | 预算三键排序 + 阶段内 Auto/Manual 比例 |
| **取舍** | Launch 放弃利润换排名；Research 放弃曝光换精准；Performance 放弃探索守利润 |

---

## 锚点的意义

策略再清晰，执行时还是会遇到：出价出多少？这个词投不投？预算够不够？

没有锚点，答案是"看情况" — 依赖经验和拍脑袋，不同时间、不同人标准漂移。

**锚点 = 目标函数在执行层的投影。** 把顶层目标（maximize 利润）压缩成一个每天能用的数字，让所有决策共享同一套标准：超过这条线亏钱，低于这条线少赚，逼近这条线最优。

---

## 核心公式

```
Max CPC = Price × Target ACoS × CVR
```

- **Price**：产品售价（产品决定）
- **Target ACoS**：目标广告成本率（老板决定，从盈亏线推导）
- **CVR**：该关键词的转化率（市场反馈，按关键词各不相同）

三层信息压成一个数：**每次点击最多值多少钱**。

---

## 为什么 Max CPC 是唯一锚点

Amazon PPC 是关键词级别的拍卖。每个关键词你做两个决定：**投不投**和**出多少钱**。

盈利约束只有一条：

```
CPC / (Price × CVR) ≤ Target ACoS
```

解出 CPC 的上界：

```
CPC ≤ Price × CVR × Target ACoS = Max CPC
```

一个不等式，一个上界。这个上界同时回答了两个决定：

- **Max CPC > 市场 CPC → 投**（有利润空间）
- **Max CPC = 0 → 不投**（CVR 为零，纯烧钱）
- **Bid = Max CPC × 60-80% → 出多少**

"找关键词"本质上是在发现每个词的 CVR，从而算出它的 Max CPC，确认是否值得参与这场拍卖。

所有其他指标要么是 Max CPC 的输入，要么是它的结果：

| 指标 | 关系 | 为什么不能做锚点 |
|------|------|-----------------|
| Price, 成本 | 输入 | 全产品固定，不区分关键词 |
| Target ACoS | 输入 | 比率，不是拍卖的单位 |
| CVR | 输入 | 只是三个变量之一 |
| Bid | 执行 | 是你的选择，不是基准 |
| 实际 CPC | 结果 | 市场决定，不可控 |
| ACoS | 结果 | 事后算出来的，不可直接设定 |

```
Max CPC ≥ Bid（你实际出的价）≥ CPC（实际扣费）
```

Max CPC 是天花板，Bid 是策略选择，CPC 是市场结果。

---

## 每个关键词有自己的 Max CPC

Price 和 Target ACoS 全产品固定，唯一变量是**每个关键词的 CVR**。

CVR = 该关键词的订单数 / 点击数，直接来自广告报表。

### 数据量决定可信度

| 点击数 | CVR 可信度 | 处理方式 |
|--------|-----------|---------|
| < 10 | 不可信 | 不动，继续积累数据 |
| 10-30 | 有参考 | 用品类平均 CVR 做保底估算 |
| 30+ | 可决策 | 直接算 Max CPC，做出价调整 |
| 50+，0 单 | CVR ≈ 0 | Max CPC = 0，否定该词 |

不是先算好再投，是**投着算、算着调**。

---

## Max CPC 的盲区

Max CPC 只算单次点击的盈亏，不算广告对自然排名的投资回报。

Amazon 广告有一个动态效应：广告花钱 → 推销量 → 提自然排名 → 自然销售增长 → 对广告的依赖降低。这就是飞轮。Max CPC 看不见飞轮。

所以需要 **TACoS（广告支出 / 总销售额）** 作为长期视角的补充：

| 锚点 | 管什么 | 时间视角 | 粒度 |
|------|--------|---------|------|
| **Max CPC** | 每个关键词赚不赚 | 短期（本周） | 关键词级 |
| **TACoS 趋势** | 飞轮转没转 | 长期（月度） | 产品级 |

两者配合使用：
- **TACoS 下降** → 飞轮在转，即使部分词 CPC > Max CPC 也可容忍（在买排名）
- **TACoS 持平或上升** → 飞轮没转，必须严格按 Max CPC 控制，不能继续烧钱买量
- **Launch 阶段**故意把 Target ACoS 放宽到盈亏线，等于主动提高 Max CPC，用短期亏损换长期排名

## 前置门槛：Listing 就绪

Max CPC = Price × Target ACoS × **CVR**。如果 Listing 没准备好（主图差、无 Review、描述烂），CVR 从根上就低，算出来的 Max CPC 很低，怎么调 Bid 都没用。

**Listing 就绪不是优化项，是开广告的前提条件。** CVR 低于品类基准时，正确的动作不是调广告，是停广告、修 Listing：

- 关键词埋入标题和 Bullet Points
- 主图符合品类点击率基准
- 至少 15+ Reviews（或使用 Vine）
- A+ 内容就绪
- 价格在竞争区间内

达标后再开广告，否则广告预算 = 给亚马逊交学费。

---

## 三个杠杆

Max CPC 只能被三个东西改变，出价本身不是杠杆：

| 杠杆 | 谁控制 | 什么时候动 |
|------|--------|-----------|
| **Price** | 产品/市场 | 竞品调价、旺季、库存压力 |
| **Target ACoS** | 老板 | 要速度（放宽）还是要利润（收紧） |
| **CVR** | Listing 质量 | 主图、标题、A+、评论、价格 |

---

## 执行体系

所有运营动作分四步：**算 → 判 → 做 → 改上游**。

### 算（每周）

从广告报表拿到每个关键词的 clicks 和 orders → 算 CVR → 算 Max CPC → 算差值（Max CPC − 实际 CPC）。

### 判（按差值分桶）

| 差值 | 含义 | 下一步 |
|------|------|--------|
| 正且大 | 利润厚 | 加量 |
| 正且小 | 微利 | 维持 |
| 负 | 在亏 | 降 Bid 或否定 |
| Max CPC = 0 | 纯烧钱 | 立刻否定 |
| 数据不够 | 不确定 | 等，继续积累点击 |

### 做（7 个动作）

| # | 动作 | 触发 | 操作 |
|---|------|------|------|
| 1 | **调 Bid** | 差值存在 | Bid = Max CPC × 60-80% |
| 2 | **否定** | 50+ clicks，CVR = 0 | 来源广告组加 Negative |
| 3 | **毕业** | CVR 高，30+ clicks 可信 | Auto → Broad → Exact，来源加否定防抢量 |
| 4 | **降级/暂停** | Exact 词连续 3 周 CPC > Max CPC | 先降 Bid，仍超标则 Pause |
| 5 | **挪预算** | 不同词差值不同 | 差值最大的词给最多预算 |
| 6 | **换匹配** | 同一词不同匹配下 CVR 不同 | 预算往 CVR 高（Max CPC 高）的匹配挪 |
| 7 | **被动发现** | Auto 跑出新成交词 | 进入"算"流程，积累数据 |
| 8 | **主动调研** | 定期用工具/竞品反查/ABA 搜索词 | 投入 Auto 或 Broad，积累数据 |

### 改上游（触发式，不是每周做）

当 Max CPC 本身需要变化时，改的是公式里的三个输入，不是 Bid：

| 动作 | 改的变量 | 触发时机 |
|------|---------|---------|
| 优化 Listing | CVR ↑ → Max CPC ↑ | CVR 低于品类基准 |
| 调价格 | Price ↑↓ → Max CPC ↑↓ | 竞品调价、旺季、库存压力 |
| 调 Target ACoS | Target ACoS ↑↓ → Max CPC ↑↓ | 老板决定要速度还是利润 |

## 对 OpenClaw 的产品意义

系统底层跑这一个公式，翻译成老板能看懂的信号卡片：

| 触发条件 | 信号 |
|---------|------|
| 实际 CPC > Max CPC，持续 2+ 周 | P0："这个词每周亏 $47，建议暂停" |
| 50+ 点击，CVR = 0 | P0："零成交词消耗 $184，立刻否定" |
| CVR 突降导致 Max CPC 大幅下移 | P1："转化率掉了，检查 Listing" |
| Max CPC − 实际 CPC 差值 > 50% | good："这个词利润厚，建议加预算" |

老板不需要理解公式，只需看到金额然后点确认/拒绝。
