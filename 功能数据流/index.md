## 系统层次结构

```
┌─ 左侧导航栏（全局持久）─────────────────────────────────────────┐
│  账号总览页                                                      │
│  ├── 账号级 KPI 汇总                                            │
│  └── 各品类健康状态列表                                          │
│                                                                  │
│  Chat（全局页）                                                  │
│  ├── 多 Session：用户可建立多个对话，历史持久化，随时切换          │
│  ├── 每个 Session 独立运行 Claude Agent SDK Loop                 │
│  ├── 工具调用通过 Skill 插件化注册（内置 Amazon Ops Skill）        │
│  └── 可访问全部已上传报表，通过自然语言提问跨品类/跨ASIN分析      │
│                                                                  │
│  品类视图页（如：床垫 / 充气泵 / 电动滑板车）                        │
│  ├── KPI 汇总（品类内所有ASIN）                                  │
│  ├── 每日告警（品类内所有ASIN）                                  │
│  ├── 广告优化行动清单（品类内所有ASIN）                          │
│  └── 库存健康看板（品类内所有ASIN）                              │
└──────────────────────────────────────────────────────────────────┘
                                    ┌─ 右侧 Context 面板（全局持久）┐
                                    │  所有页面可见，可收起          │
                                    │  镜像 context/ 文件夹内容      │
                                    │  支持上传 / 删除文件           │
                                    └───────────────────────────────┘
```

> **设计原则**：
> - Chat 是全局入口，不绑定品类；品类视图的四个 Tab 是结构化看板，数据范围始终是当前品类内的所有 ASIN。
> - Context 面板是全局文件管理层，不属于任何页面，所有功能的数据来源均从 `context/` 文件夹读取。

## Context 文件夹

`context/` 是系统的唯一数据来源。所有 Nordhive 导出文件上传后存入此目录，所有功能模块从此读取数据。

```
context/
├── product_2026-04-09.xlsx       ← 产品报表（每日更新）
├── keyword_2026-04-10.xlsx       ← 关键词监控
├── inventory_2026-04-08.xlsx     ← 库存报表
├── search_terms_30d.xlsx         ← 搜索词重构
├── campaign_3m.xlsx              ← 广告活动重构
├── us_campaign_30d.xlsx          ← US广告活动
├── placement_us_30d.xlsx         ← 广告位报表
├── cost_mgmt.xlsx                ← 成本管理
└── aba_search.xlsx               ← ABA搜索词
```

**Context 面板**（右侧可收起侧栏，所有页面持续可见）：
- 实时显示 `context/` 文件夹内容、上传日期、新鲜度状态
- 用户可直接在面板内添加 / 删除文件
- 替代"数据源新鲜度"页面级展示，成为全局文件管理入口

## 数据流总览

```
[Nordhive 导出 / Seller Central 导出]
         │
         ▼
[context/ 文件夹]  用户通过 Context 面板上传 → 文件落地存储
         │
         ├──→ [结构化面板]  解析 → 提取字段 → 计算比率 → KPI/告警/广告/库存 Tab
         │         ↑
         │    [07-配置参数]  全局默认 → 品类覆盖 → 阶段覆盖（阈值注入）
         │
         └──→ [Chat]  多 Session 持久化 → Agent SDK Loop → Skill 工具集按需查询数据
```

## 已知数据缺口

| 手册要求 | 字段来源 | 状态 | 处理方案 |
|---------|---------|------|---------|
| Buy Box 占比（每日必看） | Seller Central 业务报告 | ❌ Nordhive产品报表无此字段 | MVP用`OCR`（订单转化率）作间接替代；精确数据需从SC额外下载Business Report |
| ABA 搜索词（每月必看） | `US_ABA_Search_Compare_*.xlsx` | ⚠️ 有文件但无独立功能模块 | 作为Chat工具调用数据源（`get_file_data('aba_search')`） |
| 品类映射 | 产品报表/库存报表`产品线`字段 | ⚠️ 字段存在但填写不完整（仅床垫2个ASIN有值） | MVP 通过 seed.ts 配置，运营提需求给开发执行；v2 规划管理页面（支持产品线字段预填 + 手动分配） |

## 存储字段原则

- **存原始计数，不存比率**：`ad_spend`、`ad_sales`、`clicks`、`impressions`、`orders`、`gmv`
- **比率由系统计算**：`ACOS = ad_spend / ad_sales`、`CTR = clicks / impressions`、`CVR = orders / clicks`
- **时间序列**：产品报表按 `asin × date` 逐日累积，不覆盖
- **快照表**：搜索词、广告活动、库存、成本按最新版本覆盖存储，带 `snapshot_date` 字段
